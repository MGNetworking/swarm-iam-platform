## Décisions clés prises

- **Décision** : `core.filemode false` appliqué localement
  - **Justification** : Le projet est sur NTFS (`/mnt/d/`), WSL2 attribue le bit exécutable à tous les fichiers, git voyait 24 faux "modifiés" avec 0 diff de contenu.
  - **Alternatives rejetées** : Aucune — c'est la solution standard pour WSL2 + NTFS.

- **Décision** : Supprimer la branche `nas`
  - **Justification** : Identique à `master` (même commit `167fea6`), sans utilité claire.
  - **Alternatives rejetées** : La garder comme branche d'environnement — inutile en mono-environnement.

- **Décision** : Stratégie GitFlow avec `main` + `develop` (sans branche `prod`)
  - **Justification** : L'utilisateur proposait `main` + `prod` + `develop`. `prod` est redondante avec `main` dans un HomeLab mono-environnement. `main` = production + Release Please. `prod` réintroduire si multi-environnements un jour.
  - **Alternatives rejetées** : GitHub Flow (trop simple pour les objectifs), `main`+`prod`+`develop` (trop de merges redondants).

- **Décision** : Créer `CLAUDE.md` (contexte technique pour Claude) ET `OBJECTIVES.md` (suivi des objectifs projet, évolutif)
  - **Justification** : Séparer le contexte immuable (conventions, architecture) du pilotage évolutif (roadmap, priorités).
  - **Alternatives rejetées** : Tout dans CLAUDE.md — mélange contexte et roadmap.
